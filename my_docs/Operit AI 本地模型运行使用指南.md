## 1. 概述

### 1.1 双引擎架构

Operit AI 采用**双引擎架构**支持本地模型推理：

#### MNN 引擎（阿里巴巴轻量推理框架）
- **优势**：
  - ✅ 支持多种 GPU 后端（Vulkan/OpenCL/OpenGL）
  - ✅ Transformer 融合优化
  - ✅ 低内存优化
  - ✅ 快速推理速度
- **适用场景**：轻量模型（< 3B 参数）
- **支持格式**：MNN 格式

#### Llama.cpp 引擎（社区主流 GGUF 推理引擎）
- **优势**：
  - ✅ 支持 GGUF 格式（社区最广泛）
  - ✅ Flash Attention 优化
  - ✅ GPU 层卸载（Offloading）
  - ✅ 支持大模型
- **适用场景**：中大模型（3B-70B 参数）
- **支持格式**：GGUF 格式

### 1.2 功能特性

- ✅ **流式输出**：边思考边输出，类似在线 AI
- ✅ **工具调用**：支持调用 Operit AI 的所有工具
- ✅ **多模态**：支持图片理解（需要多模态模型）
- ✅ **角色卡**：兼容所有角色卡系统
- ✅ **上下文管理**：智能总结、记忆压缩
- ✅ **并发控制**：防止多个请求冲突

## 2. 支持的模型格式

### 2.1 GGUF 格式（推荐）

**GGUF**（GGML Universal Format）是目前最流行的本地模型格式。

#### 支持的量化等级

| 量化等级 | 大小（7B 模型） | 质量 | 速度 | 推荐场景 |
|----------|----------------|------|------|----------|
| Q2_K | ~2.8 GB | ⭐⭐ | ⭐⭐⭐⭐ | 极低内存设备 |
| Q3_K_S | ~3.2 GB | ⭐⭐⭐ | ⭐⭐⭐⭐ | 内存受限 |
| Q3_K_M | ~3.5 GB | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 平衡选择 |
| Q4_K_M | ~4.2 GB | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | **最推荐** |
| Q4_K_S | ~4.0 GB | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 速度优先 |
| Q5_K_M | ~4.8 GB | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 高质量 |
| Q5_K_S | ~4.6 GB | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 高速高质量 |
| Q6_K | ~5.5 GB | ⭐⭐⭐⭐⭐⭐ | ⭐⭐ | 极高质量 |
| Q8_0 | ~6.8 GB | ⭐⭐⭐⭐⭐⭐⭐ | ⭐ | 接近原始质量 |
| F16 | ~13 GB | ⭐⭐⭐⭐⭐⭐⭐⭐ | ⭐ | 研究用途 |
| F32 | ~26 GB | ⭐⭐⭐⭐⭐⭐⭐⭐⭐ | ⭐ | 完整精度 |

**推荐**：
- **手机/平板**：Q4_K_M 或 Q4_K_S（平衡质量和速度）
- **高性能设备**：Q5_K_M 或 Q6_K（更高质量）
- **低内存设备**：Q3_K_M 或 Q2_K（节省内存）

#### 获取 GGUF 模型

**方式 A：从 Hugging Face 下载**
```
https://huggingface.co/TheBloke
https://huggingface.co/MaziyarPanahi
https://huggingface.co/Bartowski
```

**方式 B：从 ModelScope 下载**
```
https://modelscope.cn/models
```

**方式 C：自己转换**
- 从 Safetensors 转换为 GGUF
- 使用 `convert-hf-to-gguf.py` 脚本

### 2.2 MNN 格式

**MNN** 是阿里巴巴推出的轻量推理格式。

#### 优势
- 推理速度快
- 内存占用低
- 支持多种 GPU 后端

#### 获取 MNN 模型

**方式 A：从官方转换**
```bash
# 使用 gguf2mnn.py 转换
python3 gguf2mnn.py model.gguf output.mnn
```

**方式 B：直接下载**
- MNN 官方模型库
- 阿里巴巴魔搭社区

## 3. 系统要求

### 3.1 硬件要求

#### 最低配置
- **CPU**：ARM64 架构（骁龙 660 或同等）
- **内存**：4 GB RAM
- **存储**：10 GB 可用空间
- **系统**：Android 10+

#### 推荐配置
- **CPU**：骁龙 865/天玑 1000+ 或更高
- **内存**：8 GB RAM
- **存储**：50 GB 可用空间
- **GPU**：支持 Vulkan/OpenCL
- **系统**：Android 12+

#### 性能配置（运行大模型）
- **CPU**：骁龙 8 Gen 2/天玑 9200 或更高
- **内存**：12-16 GB RAM
- **存储**：100 GB+ 可用空间
- **GPU**：支持 Vulkan 1.3+
- **散热**：主动散热或良好被动散热

### 3.2 模型大小与内存需求

| 模型参数量 | 最小内存（Q4） | 推荐内存（Q4） | 最大内存（F16） |
|------------|---------------|---------------|----------------|
| 1.5B | 2 GB | 3 GB | 4 GB |
| 3B | 3 GB | 4 GB | 8 GB |
| 7B | 5 GB | 8 GB | 16 GB |
| 13B | 9 GB | 12 GB | 32 GB |
| 34B | 22 GB | 28 GB | 68 GB |
| 70B | 45 GB | 56 GB | 140 GB |

**注意**：
- 实际内存需求 = 模型权重 + 上下文缓存 + 系统开销
- 上下文缓存：约 1MB/1K tokens
- 建议保留 20% 内存余量

## 4. 快速开始

### 4.1 一分钟快速上手

#### 步骤 1：下载预转换模型

访问 Hugging Face 下载 Q4_K_M 量化的 7B 模型：
```
TheBloke/Mistral-7B-Instruct-v0.2-GGUF
文件：mistral-7b-instruct-v0.2.Q4_K_M.gguf
大小：~4.2 GB
```

#### 步骤 2：传输到手机

```bash
# 通过 ADB 传输
adb push mistral-7b-instruct-v0.2.Q4_K_M.gguf /sdcard/Download/Operit/models/

# 或直接复制到手机
/sdcard/Download/Operit/models/
```

#### 步骤 3：在 Operit AI 中配置

1. 打开 **Operit AI** App
2. 进入 **设置** → **模型管理** → **本地模型**
3. 点击 **"+"** 添加模型
4. 选择刚才传输的 `.gguf` 文件
5. 填写模型信息：
   - 名称：Mistral 7B Instruct
   - 描述：Mistral AI 的 7B 指令模型
   - 上下文长度：4096
6. 点击 **保存**

#### 步骤 4：开始对话

1. 打开 **AI 对话**
2. 点击模型选择器（当前显示模型名称）
3. 选择 **Mistral 7B Instruct**
4. 开始对话！

### 4.2 使用内置模型下载器

## 6. 模型转换

### 6.1 Safetensors 转 GGUF

如果你有原始模型（Safetensors 格式），可以转换为 GGUF。

#### 使用 llama.cpp 转换

```bash
# 1. 克隆 llama.cpp
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp

# 2. 安装依赖
pip install -r requirements.txt

# 3. 转换模型
python convert-hf-to-gguf.py \
  /path/to/huggingface/model \
  --outfile model.gguf \
  --outtype f16

# 4. 量化（可选）
./quantize model.gguf model.Q4_K_M.gguf Q4_K_M
```

#### 转换参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--outfile` | 输出文件名 | `model.gguf` |
| `--outtype` | 输出类型 | `f16`, `f32`, `q8_0` |
| `--vocab-type` | 词表类型 | `spm`, `bpe` |

### 6.2 GGUF 转 MNN

如果需要转换为 MNN 格式：

```bash
# 使用 MNN 官方工具
python3 gguf2mnn.py model.gguf output.mnn

# 或使用 MNNConvert
MNNConvert -f GGUF \
  --modelFile model.gguf \
  --MNNModel output.mnn
```

### 6.3 量化模型

#### 使用 llama.cpp 量化

```bash
# 量化到 Q4_K_M
./quantize input.gguf output.Q4_K_M.gguf Q4_K_M

# 量化到 Q5_K_M
./quantize input.gguf output.Q5_K_M.gguf Q5_K_M

# 量化到 Q8_0
./quantize input.gguf output.Q8_0.gguf Q8_0
```

#### 量化等级选择

| 代码 | 说明 | 质量损失 | 速度提升 |
|------|------|----------|----------|
| Q2_K | 极低比特量化 | 高 | 极快 |
| Q3_K_S | 3bit 小 | 中 | 很快 |
| Q3_K_M | 3bit 中 | 中低 | 快 |
| Q4_K_S | 4bit 小 | 低 | 很快 |
| Q4_K_M | 4bit 中 | 很低 | 快 |
| Q5_K_S | 5bit 小 | 极低 | 中等 |
| Q5_K_M | 5bit 中 | 极低 | 中等 |
| Q6_K | 6bit | 几乎无 | 较慢 |
| Q8_0 | 8bit | 无 | 慢 |

## 7. 配置本地模型

### 7.1 添加模型到 Operit AI

#### 方式 A：通过界面添加

1. **打开 Operit AI**
2. **进入设置** → **模型管理**
3. **点击"+"** 或 **"添加模型"**
4. **选择模型文件**
   - 浏览到 `/sdcard/Download/Operit/models/`
   - 选择 `.gguf` 或 `.mnn` 文件
5. **填写模型信息**：
   ```
   名称：Mistral 7B Instruct
   描述：Mistral AI 的 7B 指令模型（可选）
   上下文长度：4096
   推理引擎：Llama.cpp（GGUF 格式）或 MNN
   ```
6. **点击保存**

#### 方式 B：自动扫描

1. 将模型文件放入正确目录：
   ```
   /sdcard/Download/Operit/models/
   ```
2. 打开 Operit AI
3. 设置 → 模型管理 → 扫描本地模型
4. 自动识别并添加所有模型

### 7.2 模型配置参数

#### 基础配置

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **名称** | 模型显示名称 | Mistral 7B Instruct |
| **描述** | 模型描述（可选） | 7B 参数指令模型 |
| **上下文长度** | 最大上下文窗口 | 4096 / 8192 |
| **推理引擎** | MNN 或 Llama.cpp | 根据格式自动选择 |

#### 高级配置

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **GPU 层数** | 卸载到 GPU 的层数 | 根据显存调整 |
| **线程数** | CPU 推理线程数 | 4-8 |
| **批处理大小** | 批处理 token 数 | 512 |
| **提示缓存** | 启用提示缓存 | ✅ 开启 |

## 8. 运行本地模型

### 8.1 选择模型

#### 方式 A：在对话中选择

1. 打开 **AI 对话**
2. 点击底部模型选择器（显示当前模型名称）
3. 从列表中选择本地模型
4. 开始对话

#### 方式 B：设置为默认

1. 设置 → 模型管理
2. 找到要设置的模型
3. 点击右侧菜单（⋮）
4. 选择 **设为默认**

### 8.2 开始对话

#### 步骤

1. **打开 AI 对话界面**
2. **选择本地模型**
3. **输入消息**：
   ```
   你好，请介绍一下你自己
   ```
4. **点击发送**
5. **等待模型响应**（首字延迟约 1-3 秒）
6. **流式输出**（类似在线 AI）

#### 示例对话

```
用户：你好，请介绍一下你自己

AI：你好！我是 Operit AI，一个运行在本地设备上的智能助手。
     我使用 {模型名称} 模型，可以在没有网络连接的情况下与你对话。
     我可以帮助你回答问题、编写代码、分析文档等。有什么我可以帮助你的吗？
```

### 8.3 使用高级功能

#### 工具调用

本地模型同样可以调用 Operit AI 的工具：

```
用户：帮我用 QQ 发一条消息给朋友

AI：好的，我将为您调起 QQ 发送消息。
     [调用工具：qq_send_message]
     请告诉我您要发送什么内容？
```

#### 多轮对话

本地模型支持完整的上下文记忆：

```
用户：我喜欢吃苹果
AI：好的，我记住了您喜欢吃苹果。

用户：那我喜欢吃什么水果？
AI：您喜欢吃苹果。
```

#### 角色卡

可以使用任何角色卡：

1. 点击顶部角色卡图标
2. 选择角色卡
3. 模型会以该角色身份对话

### 8.4 性能监控

#### 查看推理状态

在对话过程中，可以看到：
- **输入 Token 数**：用户消息长度
- **输出 Token 数**：AI 响应长度
- **推理速度**：tokens/秒
- **内存使用**：当前内存占用

#### 优化建议

如果速度慢：
- ✅ 降低量化等级（Q4 → Q3）
- ✅ 减少上下文长度
- ✅ 增加 GPU 层数
- ✅ 减少线程数（避免 CPU 过载）

如果内存不足：
- ✅ 使用更低量化等级
- ✅ 选择更小模型
- ✅ 减少上下文缓存

## 9. 性能优化

### 9.1 GPU 加速配置

#### Vulkan 后端（推荐）

**优势**：
- 速度快
- 功耗低
- 兼容性好

**配置方法**：
1. 设置 → 推理设置 → GPU 后端
2. 选择 **Vulkan**
3. 调整 GPU 层数

**GPU 层数推荐**：

| 设备显存 | 推荐层数（7B 模型） | 推荐层数（13B 模型） |
|----------|-------------------|-------------------|
| 2 GB | 10-15 层 | 5-10 层 |
| 4 GB | 20-25 层 | 15-20 层 |
| 6 GB | 30-32 层 | 25-30 层 |
| 8 GB+ | 全部层 | 32+ 层 |

#### OpenCL 后端

**适用场景**：
- 不支持 Vulkan 的老设备
- Mali GPU 设备

**配置**：
1. 设置 → 推理设置 → GPU 后端
2. 选择 **OpenCL**

#### OpenGL 后端

**适用场景**：
- 其他后端不可用时
- 作为备用方案

### 9.2 CPU 优化

#### 线程数配置

**推荐值**：
- **骁龙 865/870**：6 线程
- **骁龙 888/8 Gen 1**：8 线程
- **骁龙 8 Gen 2/3**：8 线程
- **天玑 9000+**：8 线程

**调整方法**：
1. 设置 → 推理设置 → CPU 线程数
2. 选择合适的线程数

**注意**：
- 线程数不是越多越好
- 过多线程会导致上下文切换开销
- 建议从 6 开始测试

#### 批处理优化

## 10. 故障排查

### 10.1 常见问题诊断

#### 问题 A：模型加载失败

**症状**：
- 点击模型后闪退
- 提示"加载失败"
- App 卡死

**可能原因**：
1. 模型文件损坏
2. 内存不足
3. 格式不支持
4. 文件路径错误

**解决方案**：
```bash
# 1. 验证文件完整性
md5sum model.gguf

# 2. 检查文件大小
ls -lh /sdcard/Download/Operit/models/model.gguf

# 3. 重新下载模型
# 4. 确保有足够内存（至少模型大小的 1.5 倍）
# 5. 确认文件格式（GGUF 或 MNN）
```

#### 问题 B：推理速度极慢

**症状**：
- 每秒 < 1 token
- 设备发热严重
- 响应延迟 > 10 秒

**解决方案**：
1. **降低量化等级**
   - Q6 → Q4
   - Q4 → Q3

2. **增加 GPU 卸载**
   - 设置 → GPU 层数 → 增加

3. **减少线程数**
   - 8 线程 → 6 线程

4. **开启低内存模式**

5. **检查散热**
   - 移除手机壳
   - 使用散热背夹

#### 问题 C：内存不足崩溃

**症状**：
- 对话中途崩溃
- 提示"Out of Memory"
- App 自动关闭

**解决方案**：
1. **使用更小模型**
   - 13B → 7B
   - 7B → 3B

2. **降低量化等级**
   - Q5 → Q4
   - Q4 → Q3

3. **减少上下文长度**
   - 8192 → 4096
   - 4096 → 2048

4. **关闭其他应用**
   - 清理后台
   - 释放内存

5. **开启低内存模式**

#### 问题 D：输出乱码或重复

**症状**：
- 输出乱码
- 重复相同内容
- 逻辑混乱

**可能原因**：
1. 模型文件损坏
2. 量化等级过低
3. 模型与格式不匹配
4. 词表加载失败

**解决方案**：
1. **重新下载模型**
2. **提高量化等级**（Q2 → Q4）
3. **确认模型格式**
4. **清除缓存**

- 快速切换模型
- 使用不同对话保存不同模型

**版本管理**：
- 建议保留旧版本作为备份
